\section{Project Overview}

\subsection{Project Title}
\textbf{The Elevated Navigator} — A Travel Itinerary Planner built on the MEAN Stack.

\subsection{Objective}
The objective of this project is to design and develop a full-stack web application that allows users to browse, book, and manage travel itineraries. The application demonstrates all nine practicals of the MEAN Stack course by integrating MongoDB, Express.js, Angular, and Node.js into a single, cohesive production-ready system.

\subsection{Scope}
\begin{itemize}
    \item User registration and login with JWT-based authentication
    \item Role-based access control (User, Admin, Superadmin)
    \item Full CRUD management of travel itineraries
    \item Responsive Angular Single-Page Application (SPA) frontend
    \item RESTful API backend with Express and Mongoose
    \item Premium UI design system using Tailwind CSS
\end{itemize}

\section{System Architecture}

The application follows a three-tier architecture:

\begin{enumerate}
    \item \textbf{Presentation Tier} — Angular SPA running in the browser at \texttt{localhost:4200}
    \item \textbf{Application Tier} — Express.js REST API running at \texttt{localhost:5000}
    \item \textbf{Data Tier} — MongoDB database accessed via Mongoose ODM
\end{enumerate}

\subsection{Authentication Middleware (middleware/auth.js)}

\begin{lstlisting}[language=JavaScript]
// Verify JWT on every protected request
const authenticate = async (req, res, next) => {
  const token = req.headers['authorization']?.split(' ')[1];
  if (!token) return res.status(401).json({ message: 'Token required' });
  const decoded = jwt.verify(token, process.env.JWT_SECRET);
  req.user = await User.findById(decoded.id).select('-password');
  next();
};

// Role-based guard
const authorize = (...roles) => (req, res, next) => {
  if (!roles.includes(req.user.role))
    return res.status(403).json({ message: 'Insufficient permissions' });
  next();
};
\end{lstlisting}

\subsection{Auth Service (auth.service.ts)}

\begin{lstlisting}[language=JavaScript]
@Injectable({ providedIn: 'root' })
export class AuthService {
  currentUser = signal<AuthUser | null>(this.loadUser());

  login(email: string, password: string) {
    return this.http.post('/api/auth/login', { email, password }).pipe(
      tap(res => {
        localStorage.setItem('token', res.token);
        this.currentUser.set(res.user);
      })
    );
  }

  logout() {
    localStorage.removeItem('token');
    this.currentUser.set(null);
    this.router.navigate(['/login']);
  }
}
\end{lstlisting}

\subsection{HTTP Interceptor (auth.interceptor.ts)}

\begin{lstlisting}[language=JavaScript]
export const authInterceptor: HttpInterceptorFn = (req, next) => {
  const token = inject(AuthService).token;
  if (token) {
    req = req.clone({
      setHeaders: { Authorization: `Bearer ${token}` },
    });
  }
  return next(req);
};
\end{lstlisting}

\subsection{Route Guard (auth.guard.ts)}

\begin{lstlisting}[language=JavaScript]
export const authGuard: CanActivateFn = () => {
  const router = inject(Router);
  if (localStorage.getItem('token')) return true;
  router.navigate(['/login']);
  return false;
};
\end{lstlisting}

\section{Authentication Flow}

\begin{enumerate}
    \item User submits email and password on the Angular login form.
    \item Angular calls \texttt{POST /api/auth/login} via \texttt{AuthService}.
    \item Express finds the user in MongoDB and runs \texttt{bcrypt.compare()}.
    \item On success, Express signs a JWT (\texttt{expiresIn: '7d'}) and returns it.
    \item Angular stores the token in \texttt{localStorage} and updates the \texttt{currentUser} signal.
    \item The HTTP interceptor attaches \texttt{Authorization: Bearer <token>} to every subsequent request.
    \item The route guard reads the token to allow or deny access to protected routes (\texttt{/dashboard}, \texttt{/admin}).
    \item On logout, the token is removed and the user is redirected to \texttt{/login}.
\end{enumerate}

\section{Conclusion}

The \textbf{Elevated Navigator} project successfully demonstrates the complete MEAN stack by building a real-world travel itinerary planning application. All nine course practicals are implemented and integrated:

\begin{itemize}
    \item The \textbf{backend} is a production-quality Express REST API with Mongoose models, JWT authentication, bcrypt password hashing, and role-based access control.
    \item The \textbf{frontend} is a modern Angular 21 SPA with standalone components, reactive signals, a functional HTTP interceptor, and a premium Tailwind CSS design system.
    \item The \textbf{database} layer uses MongoDB with well-defined Mongoose schemas including references, embedded sub-documents, and validation.
\end{itemize}

The project demonstrates how the four technologies of the MEAN stack complement each other to deliver a scalable, maintainable, and secure full-stack web application.
